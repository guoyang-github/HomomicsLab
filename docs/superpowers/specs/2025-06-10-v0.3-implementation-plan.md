## 一、本轮讨论后的最终架构共识

经过多轮质疑与修正，达成以下核心共识：

| 设计点 | 最终决策 | 关键理由 |
|--------|---------|---------|
| **Skill 内置/外部** | **执行层面完全统一** | 相同格式、相同加载器、相同执行器，来源仅作 metadata 标记 |
| **SkillDAG** | **降级为技能发现工具** | 不驱动计划生成，仅在"技能选择"时提供候选和冲突检测 |
| **GapFill** | **移除** | Agent 自写代码引用 skill 方法，比 LLM 自动生成转换代码更透明可靠 |
| **PlanEngine** | **状态驱动 + 策略模板** | 基于生信领域知识和数据状态，不依赖 SkillDAG 遍历 |
| **Agent 体系** | **1 个 AnalystAgent + 按需 Specialist** | 不是蜂群，也不是固定身份硬编码类 |
| **Agent 执行方式** | **写代码 + 引用 skill 方法** | 像人类分析师一样灵活，代码透明可审计 |
| **Reproducibility** | **记录 Agent 完整代码 + 环境锁定** | Agent 代码本身就是最好的可复现性保障 |
| **Stability** | **Schema 验证 + 版本锁定 + 回归测试** | 四层防御，保障分析稳定性 |

### 4.2.1 PlanEngine：状态驱动的计划生成

**设计原则**：
- 基于**生信领域知识**的策略模板库
- 基于**数据状态**的动态调整
- **SkillDAG 只参与"技能选择"**，不参与骨架生成

**具体实施**：

```
新增文件：
  backend/homomics_lab/agent/plan/
    ├── __init__.py
    ├── engine.py           # PlanEngine 核心
    ├── strategies.py       # 策略模板库（单细胞/空间/基因组...）
    ├── data_state.py       # 数据状态机
    └── models.py           # PlanResult, Phase, PlannedGap 等
```

**核心流程**：

```python
class PlanEngine:
    async def plan(self, intent: UserIntent, data_state: DataState) -> PlanResult:
        # 1. 选择策略（基于意图）
        strategy = self.strategies.select(intent)
        
        # 2. 生成骨架（预定义的 phase 序列）
        skeleton = strategy.generate_skeleton()
        # e.g., [QC, NORMALIZATION, DIM_REDUCTION, CLUSTERING, ANNOTATION, DE, VIZ]
        
        # 3. 根据数据状态调整
        adjusted = self._adjust_by_data_state(skeleton, data_state)
        # e.g., 检测到多批次 → 插入 BATCH_CORRECTION
        
        # 4. 为每个 phase 选择 skill（SkillDAG 辅助）
        for phase in adjusted:
            candidates = self.skill_dag.search(phase.description)
            phase.selected_skill = self._select_best_skill(candidates, data_state)
        
        # 5. 返回 PlanResult（含骨架 + 选中 skill + 预估 gap）
        return PlanResult(phases=adjusted)
```

**策略模板示例**：

```python
# strategies.py

SINGLE_CELL_STANDARD = AnalysisStrategy(
    name="single_cell_standard",
    description="Standard single-cell RNA-seq analysis pipeline",
    skeleton=[
        Phase(type=QC, required=True),
        Phase(type=NORMALIZATION, required=True),
        Phase(type=DIM_REDUCTION, required=True),
        Phase(type=CLUSTERING, required=True),
        Phase(type=ANNOTATION, required=False),
        Phase(type=DIFFERENTIAL_EXPRESSION, required=False),
        Phase(type=VISUALIZATION, required=False),
    ],
    state_checks={
        "batch_detected": InsertPhase(BATCH_CORRECTION, after=QC),
        "low_quality": InsertPhase(QC_ADVANCED, before=NORMALIZATION),
        "large_scale": ModifyParam(DIM_REDUCTION, "n_pcs", 50),
    },
)
```

### 4.2.3 AnalystAgent：代码生成与执行

**具体实施**：

  backend/homomics_lab/agent/code_generator.py # 代码生成器

**核心流程**：

```python
class AnalystAgent:
    async def execute_phase(self, phase: Phase, context: ExecutionContext) -> Result:
        # 1. 组装代码
        code = self.code_generator.generate(phase, context)
        # 代码包含：
        #   - skill 函数接口的 import 和调用
        #   - Agent 自己写的数据处理逻辑
        
        # 2. Sandbox 执行
        result = await self.sandbox.execute(code, context.workspace)
        
        # 3. 解读结果
        interpretation = await self.interpreter.interpret(phase, result)
        
        # 4. 按需 spawn Specialist
        if interpretation.needs_visualization:
            viz = VizSpecialist()
            await viz.visualize(result)
        
        return Result(data=result, interpretation=interpretation)
```

**代码生成示例**：

```python
# code_generator.py

def generate(self, phase: Phase, context: ExecutionContext) -> str:
    skill = phase.selected_skill
    
    if skill.has_function_api():
        # 生成 import + 函数调用的代码
        return f"""
import scanpy as sc
from {skill.module_path} import {skill.entry_function}

# 加载数据
adata = sc.read_h5ad("{context.input_path}")

# 调用 skill 方法
result = {skill.entry_function}(adata, **{phase.parameters})

# Agent 自己的后续处理
{phase.agent_code or ""}

# 保存结果
result.write("{context.output_path}")
"""
    else:
        # 外部 skill：生成黑盒调用包装代码
        return self._generate_blackbox_wrapper(phase, context)
```

## 十、总结

本实施方案基于多轮讨论后的最终架构共识：

1. **Skill 统一**：内置/外部执行层面无区别，来源仅作 metadata 标记
2. **SkillDAG 降级**：技能发现工具，不驱动计划生成
3. **GapFill 移除**：Agent 自写代码引用 skill 方法
4. **PlanEngine 解耦**：状态驱动 + 策略模板
5. **Agent 重构**：1 个 AnalystAgent + 按需 Specialist
6. **可复现性**：记录 Agent 完整代码 + 环境锁定
7. **稳定性**：Schema 验证 + 版本锁定 + 回归测试

**下一步**：确认本实施方案后，开始 Phase 3.1 的具体编码工作。
