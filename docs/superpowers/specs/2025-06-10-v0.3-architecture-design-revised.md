## 本轮修订的核心前提（来自用户反馈）

1. **技能数量未来远超77**，将形成开放的技能生态，不能依赖人工维护关系图谱
2. **生信分析不能完全由技能拼接而成**，中间需要LLM编辑代码进行"填缝"
3. **不对SkillDAG做明显简化**，必须解决技能调用的稳定性与准确性
4. **分析过程和结果必须可复现**，这是科研的底线要求

这四个前提彻底改变了设计重心：**从"简化可用"转向"工程严谨"**。

## 二、SkillDAG：自演化有向图（核心引擎）

### 2.1 设计原则

**不简化**。SkillDAG是PlanEngine的核心基础设施，必须支持：
- 大规模技能库（数百至数千技能）的关系管理
- 运行时自演化（边注册、验证、置信度更新）
- 多来源边融合（人工种子 + 自动推断 + 社区贡献）
- 冲突检测与消解

### 2.2 边的来源与生命周期

```
┌─────────────────────────────────────────────────────────────┐
│                    SkillDAG Edge Lifecycle                   │
├─────────────────────────────────────────────────────────────┤

│  │ Manual Seeds    │  │ Runtime Proposal│  │ Community   │ │

│  │                 │  │                 │  │             │ │
│  │ - 核心流程骨架   │  │ - 执行成功后的   │  │ - 用户提交   │ │
│  │ - 已知冲突关系   │  │   followed_by   │  │   的边验证   │ │

│                           ↓                                 │

│  ┌─────────────────────────────────────────────────────┐   │
│  │ 1. 统计验证: 至少N次成功执行 (N=5 for followed_by,   │   │

│  │ 2. 回归测试: 新增边后自动运行关联pipeline             │   │
│  │ 3. 冲突检测: 新边是否与现有confirmed边冲突            │   │

│                            │                                │
│                            └──[失败率上升]──> DEPRECATED     │
│                                                              │
│  置信度更新规则:                                              │
│  - 成功执行: +0.1 (上限1.0)                                  │
│  - 失败执行: -0.2 (下限0.0)                                  │
│  - 长期未用(30天): -0.05/天                                  │

### 2.3 数据结构

```python
from enum import Enum
from dataclasses import dataclass
from typing import Optional
from datetime import datetime


class EdgeType(str, Enum):
    """DAG边的类型。"""
    DEPENDS_ON = "depends_on"       # 硬性前置依赖
    CONFLICTS_WITH = "conflicts_with"  # 互斥（二选一）
    SPECIALIZES = "specializes"     # 专业化（更具体的版本）
    FOLLOWED_BY = "followed_by"     # 软性后续推荐
    ALTERNATIVE_TO = "alternative_to"  # 替代方案
    PRODUCES = "produces"           # 数据产出（用于schema推断）
    REQUIRES = "requires"           # 资源/环境要求


class EdgeStatus(str, Enum):
    """边的生命周期状态。"""
    CANDIDATE = "candidate"      # 候选，待验证
    CONFIRMED = "confirmed"      # 已确认，可使用
    REJECTED = "rejected"        # 验证失败，已拒绝
    DEPRECATED = "deprecated"    # 已废弃，不再使用


@dataclass
class SkillDAGEdge:
    """技能DAG中的有向边。"""
    id: str                       # 边唯一ID
    from_skill: str               # 源技能ID
    to_skill: str                 # 目标技能ID
    edge_type: EdgeType
    
    # 置信度与状态
    confidence: float = 0.5       # 0.0 ~ 1.0
    status: EdgeStatus = EdgeStatus.CANDIDATE
    
    # 来源追踪
    source: str = "unknown"       # manual_seed / runtime_proposal / schema_inference / 
                                  # history_mining / community / skill_override
    proposed_by: Optional[str] = None  # 提议者（agent_id / user_id / system）
    
    # 验证记录
    execution_count: int = 0      # 基于该边的执行次数
    success_count: int = 0        # 成功次数
    failure_count: int = 0        # 失败次数
    last_validated: Optional[datetime] = None
    
    # 上下文
    context: str = ""             # 为什么有这样的关系
    schema_compatibility_score: Optional[float] = None  # schema兼容性分数
    
    # 时间戳
    created_at: datetime = field(default_factory=datetime.now)
    updated_at: datetime = field(default_factory=datetime.now)


class SkillDAG:
    """
    自演化技能有向图。
    
    核心职责：
    1. 维护技能间的结构化关系（依赖/冲突/专业化/后续/替代）
    2. 支持多来源边融合（人工种子 + 运行时提议 + 自动推断 + 社区贡献）
    3. 运行时自演化（propose_edge / validate_edge / update_confidence）
    4. 为PlanEngine提供结构化的技能检索接口
    """
    
    def __init__(
        self,
        registry: SkillRegistry,
        db_path: Optional[Path] = None,
        manual_seed_path: Optional[Path] = None,
    ):
        self.registry = registry
        self.db_path = db_path or Path("./skill_dag.db")
        self.edges: Dict[str, SkillDAGEdge] = {}
        self._init_db()
        self._load_manual_seeds(manual_seed_path)
        self._load_from_db()
    
    # ─────────────────────────────────────────
    # 核心查询接口（供PlanEngine使用）
    # ─────────────────────────────────────────
    
    def search(
        self,
        query: str,
        top_k: int = 10,
        include_neighbors: bool = True,
        exclude_conflicts: bool = True,
        min_confidence: float = 0.5,
    ) -> List[SkillDAGSearchResult]:
        """
        结构化技能检索。
        
        返回：向量匹配结果 + DAG边邻居 + 冲突信号
        """
        # 1. 语义搜索（向量匹配）
        semantic_results = self._semantic_search(query, top_k=top_k)
        
        # 2. 图邻居扩展（如果启用）
        if include_neighbors:
            neighbors = self._expand_neighbors(semantic_results, hops=2)
            semantic_results.extend(neighbors)
        
        # 3. 过滤冲突（如果启用）
        if exclude_conflicts:
            semantic_results = self._exclude_conflicts(semantic_results)
        
        # 4. 按综合得分排序
        return self._rank_results(semantic_results, min_confidence)
    
    def get_dependencies(
        self,
        skill_id: str,
        transitive: bool = True,
        min_confidence: float = 0.7,
    ) -> List[str]:
        """获取技能的所有依赖（可选传递闭包）。"""
        ...
    
    def get_conflicts(self, skill_id: str) -> List[str]:
        """获取技能的所有冲突技能。"""
        ...
    
    def get_followed_by(
        self,
        skill_id: str,
        min_confidence: float = 0.6,
    ) -> List[Tuple[str, float]]:
        """获取技能的推荐后续技能及置信度。"""
        ...
    
    def get_alternatives(self, skill_id: str) -> List[str]:
        """获取技能的替代方案。"""
        ...
    
    def validate_plan(
        self,
        skill_sequence: List[str],
    ) -> PlanValidationResult:
        """验证技能序列是否满足DAG约束。"""
        ...
    
    # ─────────────────────────────────────────
    # 自演化接口（运行时调用）
    # ─────────────────────────────────────────
    
    def propose_edge(
        self,
        from_skill: str,
        to_skill: str,
        edge_type: EdgeType,
        context: str = "",
        proposed_by: str = "system",
    ) -> SkillDAGEdge:
        """
        提议一条新边。
        
        调用时机：
        - Agent成功执行skill A后紧接着执行skill B → propose followed_by
        - Schema兼容性检测通过 → propose depends_on / produces
        - 用户明确指示两个技能的关系 → 直接propose
        """
        edge_id = f"{from_skill}_{edge_type.value}_{to_skill}"
        
        # 检查是否已存在
        if edge_id in self.edges:
            return self.edges[edge_id]
        
        edge = SkillDAGEdge(
            id=edge_id,
            from_skill=from_skill,
            to_skill=to_skill,
            edge_type=edge_type,
            confidence=0.3,  # 初始低置信度
            status=EdgeStatus.CANDIDATE,
            source="runtime_proposal",
            proposed_by=proposed_by,
            context=context,
        )
        
        self.edges[edge_id] = edge
        self._persist_edge(edge)
        
        return edge
    
    def record_execution(
        self,
        from_skill: Optional[str],
        to_skill: str,
        success: bool,
        gap_filled: bool = False,
        gap_success: bool = True,
    ) -> None:
        """
        记录一次基于DAG边的执行结果，更新置信度。
        
        这是自演化的核心驱动力。
        """
        if from_skill is None:
            return
        
        edge_id = f"{from_skill}_followed_by_{to_skill}"
        edge = self.edges.get(edge_id)
        
        if edge is None:
            # 自动propose新边
            edge = self.propose_edge(
                from_skill=from_skill,
                to_skill=to_skill,
                edge_type=EdgeType.FOLLOWED_BY,
                context=f"Observed in execution flow, success={success}",
            )
        
        # 更新统计
        edge.execution_count += 1
        if success and (not gap_filled or gap_success):
            edge.success_count += 1
            edge.confidence = min(1.0, edge.confidence + 0.1)
        else:
            edge.failure_count += 1
            edge.confidence = max(0.0, edge.confidence - 0.2)
        
        edge.last_validated = datetime.now()
        
        # 状态迁移
        self._transition_status(edge)
        self._persist_edge(edge)
    
    def _transition_status(self, edge: SkillDAGEdge) -> None:
        """根据置信度和统计数据迁移边的状态。"""
        if edge.status == EdgeStatus.CANDIDATE:
            # 升级为CONFIRMED的条件
            min_executions = 5 if edge.edge_type == EdgeType.FOLLOWED_BY else 10
            if (edge.execution_count >= min_executions and 
                edge.success_count / edge.execution_count >= 0.8 and
                edge.confidence >= 0.7):
                edge.status = EdgeStatus.CONFIRMED
        
        elif edge.status == EdgeStatus.CONFIRMED:
            # 降级为DEPRECATED的条件
            if edge.execution_count >= 10:
                failure_rate = edge.failure_count / edge.execution_count
                if failure_rate > 0.5 or edge.confidence < 0.3:
                    edge.status = EdgeStatus.DEPRECATED
    
    # ─────────────────────────────────────────
    # 自动推断（背景任务）
    # ─────────────────────────────────────────
    
    async def infer_from_history(
        self,
        tracker: SkillPerformanceTracker,
        min_cooccurrence: int = 3,
    ) -> List[SkillDAGEdge]:
        """
        基于执行历史推断followed_by关系。
        
        从skill_executions表中挖掘：
        - 同一pipeline中skill A后紧跟skill B的频率
        - 成功执行的对形成followed_by候选
        """
        ...
    
    async def infer_from_schemas(self) -> List[SkillDAGEdge]:
        """
        基于输入输出schema兼容性推断produces/depends_on关系。
        """
        ...
```

### 2.5 人工种子（Bootstrap）

即使SkillDAG是自演化的，初始状态也需要人工种子边来启动：

```yaml
# backend/homomics_lab/skills/skill_dag_seeds.yaml
# 人工种子边：为自演化提供初始骨架
# 这些边以 CONFIRMED 状态加载，置信度 1.0

version: "0.3.0"
seeds:
  # === 单细胞标准流程骨架 ===
  - from: scanpy_qc
    to: scanpy_pca
    type: followed_by
    context: "Standard single-cell pipeline"
  
  - from: scanpy_pca
    to: scanpy_cluster
    type: followed_by
    context: "Standard single-cell pipeline"
  
  - from: scanpy_cluster
    to: scanpy_annotation
    type: followed_by
    context: "Standard single-cell pipeline"
  
  # === 硬性约束 ===
  - from: scanpy_cluster
    to: scanpy_qc
    type: depends_on
    context: "Clustering requires QC-filtered data"
  
  # === 已知冲突 ===
  - from: scanpy_cluster
    to: seurat_cluster
    type: conflicts_with
    context: "Redundant clustering on same dataset"
```

**种子边 vs 自演化边的比例**：
- 初始状态：种子边占~20%（核心骨架），其余由自演化填充
- 运行3个月后：自演化边占~80%，种子边作为"锚点"保留

## 三、GapFiller：技能间代码生成引擎（核心创新）

### 3.1 问题定义

生信分析中，技能A的输出和技能B的输入**往往不是直接兼容的**。

**典型场景**：

```
[scanpy_qc] ──► output: AnnData对象
                     ├── obs: n_genes, n_counts, pct_counts_mt
                     └── var: n_cells, highly_variable
                
                ▼  gap: 下一个技能需要特定的obs列 "batch" 来做批次校正
                
[scanpy_harmony] ──► input: AnnData with obs['batch']
```

**Gap的类型**：
1. **字段缺失**：输出缺少下游需要的字段
2. **格式转换**：Seurat对象 ↔ AnnData对象
3. **维度变换**：矩阵形状不匹配
4. **参数映射**：字段名不同（e.g., `cell_type` vs `annotation`）
5. **数据类型**：categorical vs string vs int
6. **索引对齐**：barcode格式不同

### 3.2 GapFiller架构

```python
class GapFiller:
    """
    技能间代码自动生成引擎。
    
    核心职责：弥合上游技能输出与下游技能输入之间的gap，
    通过LLM生成转换代码，并在sandbox中验证。
    """
    
    def __init__(
        self,
        llm_router: LLMRouter,
        sandbox: LocalSandbox,
        cache_dir: Path,
    ):
        self.llm_router = llm_router
        self.sandbox = sandbox
        self.cache_dir = cache_dir
        self.cache_dir.mkdir(parents=True, exist_ok=True)
    
    async def fill_gap(
        self,
        from_skill: SkillDefinition,
        to_skill: SkillDefinition,
        upstream_output: Dict[str, Any],
        workspace: WorkspaceManager,
    ) -> GapFillResult:
        """
        填补两个技能之间的gap。
        
        流程：
        1. Schema分析 → 判断gap类型
        2. 如果直接兼容 → 透传（零成本）
        3. 如果需要转换 → 生成转换代码
        4. Sandbox验证 → 确保转换正确
        5. 缓存 → 相同gap复用
        """
        
        # 1. 生成cache key
        cache_key = self._make_cache_key(from_skill, to_skill)
        
        # 2. 检查缓存
        cached = self._load_from_cache(cache_key)
        if cached and cached.validation_passed:
            return await self._apply_cached_gap(cached, upstream_output, workspace)
        
        # 3. Schema分析
        gap_analysis = self._analyze_gap(
            from_output_schema=from_skill.output_schema,
            to_input_schema=to_skill.input_schema,
            upstream_output=upstream_output,
        )
        
        if gap_analysis.gap_type == GapType.NONE:
            # 直接兼容，无需填缝
            return GapFillResult(
                code=None,
                transformed_data=upstream_output,
                gap_type=GapType.NONE,
            )
        
        # 4. 生成转换代码
        code = await self._generate_transform_code(
            gap_analysis=gap_analysis,
            from_skill=from_skill,
            to_skill=to_skill,
            upstream_output_sample=self._sample_output(upstream_output),
        )
        
        # 5. Sandbox验证
        validation = await self._validate_in_sandbox(
            code=code,
            input_data=upstream_output,
            expected_schema=to_skill.input_schema,
        )
        
        if not validation.passed:
            # 验证失败 → HITL请求人工介入
            return GapFillResult(
                code=code,
                transformed_data=None,
                gap_type=gap_analysis.gap_type,
                validation=validation,
                requires_hitl=True,
                hitl_context=f"Gap fill validation failed: {validation.error}",
            )
        
        # 6. 应用转换
        transformed = await self._apply_transform(code, upstream_output, workspace)
        
        # 7. 缓存
        self._save_to_cache(cache_key, code, validation)
        
        return GapFillResult(
            code=code,
            transformed_data=transformed,
            gap_type=gap_analysis.gap_type,
            validation=validation,
            requires_hitl=False,
        )
    
    async def _generate_transform_code(
        self,
        gap_analysis: GapAnalysis,
        from_skill: SkillDefinition,
        to_skill: SkillDefinition,
        upstream_output_sample: Dict[str, Any],
    ) -> str:
        """调用LLM生成转换代码。"""
        
        prompt = f"""You are a bioinformatics data transformation expert.
Your task is to write Python code that transforms the output of one skill into the input format expected by another skill.

## Upstream Skill: {from_skill.name}
Output Schema:
```json
{from_skill.output_schema.model_dump_json(indent=2)}
```

Sample Output:
```json
{json.dumps(upstream_output_sample, indent=2, default=str)[:2000]}
```

## Downstream Skill: {to_skill.name}
Input Schema:
```json
{to_skill.input_schema.model_dump_json(indent=2)}
```

## Gap Analysis
- Gap Type: {gap_analysis.gap_type.value}
- Missing Fields: {gap_analysis.missing_fields}
- Type Mismatches: {gap_analysis.type_mismatches}
- Description: {gap_analysis.description}

## Requirements
1. Write Python code that reads `input_data` (the upstream output) and produces `output_data` (matching downstream input schema)
2. Use scanpy/AnnData operations where appropriate
3. Handle edge cases (empty data, missing fields, type conversion)
4. Do NOT write a complete script, only the transformation function
5. The code will be executed in a sandbox with access to: numpy, pandas, scanpy, anndata

## Output Format
Return ONLY the Python code, wrapped in ```python ... ``` blocks.
"""
        
        response = await self.llm_router.generate(
            messages=[{"role": "user", "content": prompt}],
            temperature=0.2,  # 低温度确保确定性
        )
        
        # 提取代码块
        code = self._extract_code_blocks(response.content)
        return code
    
    async def _validate_in_sandbox(
        self,
        code: str,
        input_data: Dict[str, Any],
        expected_schema: SkillOutputSchema,
    ) -> GapValidationResult:
        """在sandbox中验证转换代码。"""
        
        validation_script = f"""
import json
import sys

# Load input
input_data = json.loads({repr(json.dumps(input_data, default=str))})

# Execute transform
{code}

# Validate output against schema
output_data = locals().get('output_data', locals().get('result', {{}}))

# Basic validation
validation_errors = []
schema = {repr(expected_schema.model_dump())}

# Check required fields
for field in schema.get('required', []):
    if field not in output_data:
        validation_errors.append(f"Missing required field: {{field}}")

# Check types (simplified)
for field, prop in schema.get('properties', {{}}).items():
    if field in output_data:
        expected_type = prop.get('type')
        actual = output_data[field]
        # Type checking logic...

# Result
result = {{
    "passed": len(validation_errors) == 0,
    "errors": validation_errors,
    "output_sample": str(output_data)[:1000] if output_data else None,
}}

with open('__validation_result__.json', 'w') as f:
    json.dump(result, f)
"""
        
        # 在sandbox中运行
        sandbox_result = await self.sandbox.run_python(
            code=validation_script,
            inputs={},
            timeout_seconds=30.0,
        )
        
        return GapValidationResult(
            passed=sandbox_result.get("passed", False),
            errors=sandbox_result.get("errors", []),
            output_sample=sandbox_result.get("output_sample"),
        )
```

class ReproducibilityEngine:
    """
    可复现性引擎。
    
    在分析执行的每个阶段记录完整信息，最终输出ReproducibilityBundle。
    """
    
    def __init__(
        self,
        workspace: WorkspaceManager,
        tracker: SkillPerformanceTracker,
    ):
        self.workspace = workspace
        self.tracker = tracker
        self._current_bundle: Optional[ReproducibilityBundleBuilder] = None
    
    def start_analysis(self, project_id: str, random_seed: int = 42) -> None:
        """开始一次新的分析，初始化BundleBuilder。"""
        self._current_bundle = ReproducibilityBundleBuilder(
            project_id=project_id,
            random_seed=random_seed,
            environment_lock=self._capture_environment(),
        )
    
    def record_plan(self, task_tree: TaskTree, plan_context: Dict) -> None:
        """记录生成的执行计划。"""
        self._current_bundle.set_execution_snapshot(
            ExecutionSnapshot(
                task_tree=task_tree.to_snapshot(),
                plan_version=plan_context.get("plan_engine_version"),
                plan_prompt=plan_context.get("prompt"),
                plan_llm_model=plan_context.get("llm_model"),
                timestamp=datetime.now(),
            )
        )
    
    def record_skill_execution(
        self,
        task: TaskNode,
        skill: SkillDefinition,
        result: Dict[str, Any],
    ) -> None:
        """记录技能执行。"""
        # 记录技能版本
        self._current_bundle.add_skill_version(
            skill_id=skill.id,
            version=skill.version,
            checksum=self._compute_skill_checksum(skill),
        )
        
        # 记录数据血缘
        if result.get("output_files"):
            for file_path in result["output_files"]:
                self._current_bundle.add_lineage_node(
                    LineageNode(
                        node_id=str(uuid.uuid4()),
                        path=file_path,
                        type="intermediate",
                        checksum=self._compute_file_checksum(file_path),
                        created_by_task=task.id,
                    )
                )
    
    def record_gap_fill(self, gap_result: GapFillResult) -> None:
        """记录填缝代码。"""
        if gap_result.code:
            self._current_bundle.add_gap_code(
                GapCodeEntry(
                    gap_id=gap_result.gap_id,
                    from_skill=gap_result.from_skill,
                    to_skill=gap_result.to_skill,
                    code=gap_result.code,
                    llm_model=gap_result.llm_model,
                    llm_temperature=gap_result.llm_temperature,
                    validation_result=gap_result.validation,
                    timestamp=gap_result.timestamp,
                )
            )
    
    def record_hitl_decision(
        self,
        checkpoint: HITLCheckpoint,
        choice: str,
        parameters: Dict[str, Any],
    ) -> None:
        """记录HITL决策（人工选择是分析的一部分，必须记录才能复现）。"""
        self._current_bundle.add_hitl_decision(
            HITLDecisionRecord(
                checkpoint_id=checkpoint.id,
                choice=choice,
                parameters=parameters,
                timestamp=datetime.now(),
            )
        )
    
    def finalize(self) -> ReproducibilityBundle:
        """完成分析，生成最终的Bundle。"""
        bundle = self._current_bundle.build()
        
        # 保存到workspace
        bundle_path = self.workspace.get_path(".metadata/reproducibility_bundle.json")
        bundle_path.write_text(bundle.to_json(), encoding="utf-8")
        
        # 同时保存为.homics的一部分
        return bundle
    
    async def replay(self, bundle: ReproducibilityBundle) -> ReplayResult:
        """
        重放一次分析。
        
        这是可复现性的终极验证：给定Bundle，能否得到相同结果？
        """
        # 1. 环境验证
        env_check = self._verify_environment(bundle.environment_lock)

                success=False,
                error=f"Environment incompatible: {env_check.differences}",
            )
        
        # 2. 技能版本验证
        skill_check = self._verify_skill_versions(bundle.skill_version_lock)
        if not skill_check.compatible:
            return ReplayResult(
                success=False,
                error=f"Skill versions changed: {skill_check.differences}",
            )
        
        # 3. 设置随机种子
        import random
        import numpy as np
        random.seed(bundle.random_seed)
        np.random.seed(bundle.random_seed)
        
        # 4. 重放TaskTree
        # ...（使用相同的PlanEngine配置和参数）
        
        # 5. 重放HITL决策（使用记录的选择，无需人工介入）
        # ...
        
        # 6. 重放填缝代码（使用缓存的代码，不再调用LLM）
        # ...
        
        # 7. 对比结果
        return self._compare_results(original_bundle=bundle, replayed_results=...)

class SchemaValidator:
    """严格的输入输出schema验证。"""
    
    def validate_input(
        self,
        skill: SkillDefinition,
        inputs: Dict[str, Any],

        for required in schema.required:

                errors.append(f"Missing required field: {required}")
        
        # 检查字段类型

                expected_type = prop.get("type")

                errors.append(f"Unexpected fields: {extra}")
        
        return ValidationResult(
            passed=len(errors) == 0,

class VersionLocker:
    """项目级版本锁定。"""
    
    def lock_project(self, project_id: str) -> VersionLock:
        """锁定项目使用的所有技能版本和环境。"""
        lock = VersionLock(
            project_id=project_id,
            locked_at=datetime.now(),
            skills=self._lock_skills(),
            environment=self._lock_environment(),
        )
        
        # 保存到workspace
        lock_path = self.workspace.get_path(".metadata/version.lock")
        lock_path.write_text(lock.to_json(), encoding="utf-8")
        
        return lock
    
    def _lock_skills(self) -> Dict[str, str]:
        """锁定所有已注册技能的版本。"""
        return {
            skill.id: skill.version
            for skill in self.registry.list_all()
        }
    
    def _lock_environment(self) -> str:
        """锁定Python环境。"""
        import subprocess
        result = subprocess.run(
            ["pip", "freeze"],
            capture_output=True,
            text=True,
        )
        return result.stdout
    
    def verify(self, project_id: str) -> LockVerificationResult:
        """验证当前环境是否与锁定一致。"""
        lock_path = self.workspace.get_path(".metadata/version.lock")
        if not lock_path.exists():
            return LockVerificationResult(passed=True, warnings=["No lock file found"])
        
        lock = VersionLock.from_json(lock_path.read_text())
        
        # 检查技能版本
        version_mismatches = []
        for skill_id, locked_version in lock.skills.items():
            current = self.registry.get(skill_id)
            if current and current.version != locked_version:
                version_mismatches.append(
                    f"{skill_id}: locked={locked_version}, current={current.version}"
                )
        
        return LockVerificationResult(
            passed=len(version_mismatches) == 0,
            version_mismatches=version_mismatches,
        )

class CanaryReleaser:
    """金丝雀发布：新技能/新边先在隔离环境验证。"""
    
    async def canary_release(
        self,
        skill: SkillDefinition,
        num_runs: int = 5,
    ) -> CanaryResult:
        """
        在隔离环境中运行新技能N次，观察成功率和输出稳定性。
        """
        results = []
        for i in range(num_runs):
            result = await self._run_in_isolation(skill)
            results.append(result)
        
        success_rate = sum(1 for r in results if r.success) / num_runs
        
        # 检查输出稳定性（相同输入是否产生相同输出）
        output_hashes = [r.output_hash for r in results if r.success]
        deterministic = len(set(output_hashes)) == 1
        
        return CanaryResult(
            skill_id=skill.id,
            num_runs=num_runs,
            success_rate=success_rate,
            deterministic=deterministic,
            passed=success_rate >= 0.8 and deterministic,
        )
```

## 六、PlanEngine（修订版）

### 6.1 与SkillDAG的协作

```python
class PlanEngine:
    """
    基于SkillDAG的动态计划生成引擎（修订版）。
    
    核心变化：
    1. 完全依赖SkillDAG的自演化边，不再硬编码pipeline
    2. 计划生成后必须通过StabilityGuard验证
    3. 计划本身被记录到ReproducibilityBundle
    """
    
    def __init__(
        self,
        skill_dag: SkillDAG,
        llm_router: LLMRouter,
        stability_guard: StabilityGuard,
    ):
        self.skill_dag = skill_dag
        self.llm_router = llm_router
        self.stability_guard = stability_guard
    
    async def plan(
        self,
        intent: UserIntent,
        context: PlanContext,
    ) -> PlanResult:
        """
        生成执行计划。
        
        流程：
        1. SkillDAG.search() → 候选技能子图
        2. LLM在DAG约束下生成计划
        3. StabilityGuard验证计划（schema兼容性、版本锁定）
        4. 返回TaskTree + 可复现性元数据
        """
        
        # Step 1: 候选技能检索（使用SkillDAG的结构化搜索）
        candidates = self.skill_dag.search(
            query=intent.to_natural_language(),
            top_k=15,
            include_neighbors=True,
            exclude_conflicts=True,
            min_confidence=0.5,
        )
        
        # Step 2: 构建子图
        subgraph = self._build_subgraph(candidates)
        
        # Step 3: LLM规划（在DAG约束下）
        prompt = self._build_planning_prompt(intent, subgraph, context)
        raw_plan = await self.llm_router.generate_structured(
            prompt,
            schema=RawPlanSchema,
        )
        
        # Step 4: DAG约束验证（硬约束不能违反）
        validated = self._validate_against_dag(raw_plan, subgraph)
        if not validated.valid:
            # 违反硬约束 → 修正计划或报错
            corrected = await self._correct_plan(validated, subgraph)
            validated = corrected
        
        # Step 5: StabilityGuard验证（schema兼容性）
        stability_check = await self.stability_guard.validate_plan(validated.skill_sequence)
        if not stability_check.passed:
            # Schema不兼容 → 提示需要GapFiller或人工介入
            validated = self._annotate_gaps(validated, stability_check.gaps)
        
        # Step 6: 构建TaskTree
        task_tree = self._to_task_tree(validated)
        
        # Step 7: 记录可复现性元数据
        reproducibility_context = {
            "plan_engine_version": "0.3.0",
            "prompt": prompt,
            "llm_model": self.llm_router.current_model,
        }
        
        return PlanResult(
            task_tree=task_tree,
            skill_sequence=validated.skill_sequence,
            gaps=stability_check.gaps,  # 标记需要填缝的位置
            reproducibility_context=reproducibility_context,
        )
```

### 7.1 三层架构确认（不变，但L1扩展）

**修订**：L1中的DAG边信息不再来自SKILL.md（可选），而是来自**SkillDAG的实时查询**。这样：
- 用户自建skill无需写dag_edges
- PlanEngine始终使用最新、最准确的结构化关系
- SkillDAG的自演化直接体现在L1上下文中

### 7.2 ContextAssembler修订

```python
class ContextAssembler:
    def assemble_for_planning(self) -> str:
        """为PlanEngine组装L1上下文。"""
        # 1. 所有技能的YAML frontmatter（静态）
        skill_catalog = self._build_skill_catalog()
        
        # 2. SkillDAG的confirmed边信息（动态，实时查询）
        dag_context = self._build_dag_context(
            min_confidence=0.7,  # 只包含高置信度边
            max_edges=200,       # token预算限制
        )
        
        return f"""
## Available Skills
{skill_catalog}

## Skill Relationships (confirmed, confidence >= 0.7)
{dag_context}
"""
    
    def _build_dag_context(self, min_confidence: float, max_edges: int) -> str:
        """从SkillDAG实时构建关系上下文。"""
        edges = self.skill_dag.get_confirmed_edges(
            min_confidence=min_confidence,
            limit=max_edges,
        )
        
        lines = []
        for edge in edges:
            lines.append(
                f"- {edge.from_skill} --[{edge.edge_type}]--> {edge.to_skill} "
                f"(confidence: {edge.confidence:.2f})"
            )
        return "\n".join(lines)
```

## 十一、设计哲学总结

### 11.1 本轮修订的核心转变

| 维度 | 第一版设计 | 修订版设计 | 转变原因 |
|------|-----------|-----------|---------|
| **SkillDAG** | 手动YAML + 可选覆盖 | **完整自演化有向图** | 技能数量将远超77，必须自动化 |
| **技能拼接** | 直接透传 | **GapFiller填缝引擎** | 生信分析中技能间不直接兼容是常态 |
| **稳定性** | 依赖测试 | **四层防御体系** | 科研分析不能容忍随机失败 |
| **可复现性** | 项目导出 | **ReproducibilityBundle** | 科研的底线要求 |
| **LLM角色** | 规划+执行 | **规划+填缝+解读** | LLM的核心价值是弥合结构化技能间的gap |

### 11.2 "取所长，不拼凑"的修订实践

1. **Ruflo的GOAP → DAG+LLM混合**：保留规划思想，但用SkillDAG的自演化边替代精确定义的前置条件
2. **Ruflo的Swarm → 动态角色**：不变，单机场景无需分布式协调
3. **SkillDAG论文 → 工程化自演化**：保留核心机制（propose_edge / 置信度 / 状态迁移），但降低学术benchmark要求
4. **Odysseus → 模块化Agent Loop + 本地优先**：不变
5. **CowAgent → 三层记忆 + 知识库**：不变

### 11.3 核心价值主张

v0.3.0的核心升级是**"从被动执行到主动智能，从一次性分析到可复现科研"**的质变：

- **SkillDAG**：Agent能自动学习和维护技能间的结构化关系，规模越大越智能
- **GapFiller**：Agent能自动弥合技能间的gap，不再受限于预定义的组合
- **ReproducibilityEngine**：每次分析都是完全可复现的，满足发表要求
- **StabilityGuard**：四层防御确保分析的稳定性和准确性

这四个模块的协同，将使HomomicsLab从一个"工具执行平台"进化为一个**"自适应、自演化、可复现的生物信息学智能分析基础设施"**。
