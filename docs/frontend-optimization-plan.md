# HomomicsLab 前端优化方案

## 1. 优化目标

针对当前前端“过于简化、缺乏必要配置能力、交互体验与美观度不足、功能不完善”四大问题，制定以下优化方案：

- **提升视觉质感与专业度**：从“演示原型”升级为“科研级工具”风格。
- **补齐关键配置入口**：让用户无需改代码即可配置 LLM、模型、沙箱、预算等。
- **增强交互体验**：减少点击次数、优化状态反馈、支持键盘快捷键、拖拽上传等。
- **补全功能闭环**：将后端已具备但前端未暴露的能力（计划审批、HITL、执行监控、配置中心）完整呈现。

---

## 2. 核心问题诊断

### 2.1 界面过于简化

- **问题表现**：聊天区、工作区仅为基础左右分栏，缺少品牌识别、空间层次、卡片化设计。
- **影响**：用户第一印象偏向“Demo”，难以建立对工具能力的信任。

### 2.2 缺乏必要配置能力

- **问题表现**：前端无设置入口，LLM Provider、模型选择、本地模型、沙箱后端、预算限制、语义检索模型等均不可配置。
- **影响**：用户无法根据自身环境（本地/云端、OpenAI/Claude/本地 Ollama）灵活调整， onboarding 门槛高。

### 2.3 交互体验与美观度不足

- **问题表现**：
  - 按钮、标签页、输入框样式单一，缺少悬停/聚焦/禁用态细节。
  - 消息气泡样式简陋，不支持代码块高亮、Markdown、文件预览。
  - 任务节点卡片信息密度低，缺少进度、日志、重试等操作入口。
  - 缺少加载骨架屏、错误提示、操作确认、撤销反馈。
- **影响**：用户在使用过程中容易迷失，错误发生后难以定位。

### 2.4 功能不完善

- **问题表现**：
  - 计划审批（Plan Approval）、HITL 人工介入、辩论回复等对话交互未接入 UI。
  - 任务执行状态未通过 SSE 实时推送展示。
  - Skill 管理缺少信任等级、锁定、测试运行、升级发布等操作。
  - Domain 市场缺少本地导入、导出、模板预览。
  - 可复现包（ReproducibilityBundle）下载/查看入口缺失。
- **影响**：后端能力无法被用户感知，产品价值大打折扣。

---

## 3. 设计原则

1. **科研工具感**：冷静、清晰、可信赖。主色调保留品牌蓝，但降低饱和度，增加中性灰、成功绿、警告黄、错误红的功能色。
2. **信息密度适中**：科研用户需要同时看到“对话、任务树、参数、结果”，通过分栏、折叠面板、标签页组织。
3. **渐进式披露**：初级用户只看到核心对话；高级用户可展开参数、代码、日志、配置。
4. **即时反馈**：每个操作都有加载、成功、失败、空状态反馈。
5. **键盘友好**：支持 Enter 发送、Esc 关闭、/ 唤起命令、? 查看快捷键。

## 4. 具体优化项

### 4.1 全局 UI 升级

- **引入轻量级 Design System**：
  - 统一 `Button`、`Input`、`Select`、`Modal`、`Tooltip`、`Badge`、`Skeleton`、`EmptyState` 等基础组件。
  - 在 `frontend/src/components/ui/` 下沉淀通用组件，避免各业务组件样式碎片化。
- **改进布局**：
  - 顶部应用栏：增加 Logo、项目名称、全局搜索、通知中心、用户设置入口。
  - 左侧导航：从单一“对话/工作区”扩展为“首页 / 对话 / 工作流 / 报告 / Skills / Domains / 设置”。
  - 右侧区域根据上下文动态展示：聊天、画布、详情、配置、日志。
- **支持主题切换**：默认浅色模式，可切换深色模式，适配长时间实验分析场景。

### 4.2 新增配置中心（Settings）

新增 `/settings` 页面或抽屉面板，至少包含以下配置模块：

| 模块 | 配置项 | 说明 |
|------|--------|------|
| **AI 模型** | Provider（OpenAI / Anthropic / Azure / Local） | 选择 LLM 供应商 |
| | API Key / Base URL / 模型名称 | 支持本地 Ollama 等自定义端点 |
| | Temperature / Max Tokens | 控制生成创意度与长度 |
| | Embedding 模型 | 用于语义检索与 Skill 召回 |
| **执行环境** | 沙箱后端（subprocess / docker / local） | CodeAct 代码执行环境 |
| | 工作目录 / 数据目录 | 挂载路径配置 |
| | 超时时间 / 内存限制 | 防止长任务耗尽资源 |
| **预算与成本** | 单次请求预算 / 月度预算 | 成本控制 |
| | 审批阈值 | 超过 token 或成本时触发人工确认 |
| **账号与数据** | 数据保留策略 | 自动清理历史任务 |
| | 导出/导入配置 | 便于迁移与备份 |

实现方式：
- 新增 `frontend/src/components/settings/SettingsPanel.tsx`。
- 新增 `/api/settings` 相关 API 封装，后端提供 GET/PUT 接口。
- 表单使用分组卡片，关键项变更后提示“需要重启/重新加载”或立即生效。

### 4.3 聊天体验重构

- **消息气泡升级**：
  - 支持 Markdown 渲染、LaTeX（科研公式）、代码块高亮、表格。
  - Agent 消息支持折叠/展开思考过程、展示引用的 Skills、展示生成的代码片段。
  - 增加复制、重新生成、点赞/点踩反馈按钮。
- **计划卡片**：
  - 当后端返回 Plan 时，渲染为结构化步骤卡片，支持“批准 / 修改 / 拒绝”操作。
  - 修改模式允许用户拖拽调整步骤顺序、编辑步骤描述、删除/新增步骤。
- **HITL 交互**：
  - 当任务状态为 `awaiting_human` 时，在聊天区弹出确认/选择/输入面板。
  - 支持“确认继续”、“提供额外信息”、“中止任务”。
- **快捷命令**：
  - 输入 `/` 呼出命令菜单：/analyze /generate-skill /import-domain /settings 等。
  - 支持文件拖拽上传，自动识别 CSV/TSV/FASTQ/AnnData/H5AD 等生物信息格式。
- **多会话管理**：
  - 左侧会话列表，支持新建、重命名、删除、搜索历史会话。

### 4.4 工作流/任务可视化增强

- **节点卡片信息增强**：
  - 显示任务名称、描述、状态徽章、进度条、耗时、Cost。
  - 提供“查看日志”、“查看代码”、“重试”、“跳过”、“中止”等操作按钮。
- **画布交互**：
  - 支持缩放、平移、自动布局（DAG 布局）。
  - 点击节点右侧展开详情抽屉（Detail Drawer），而非替换整个右侧面板。
  - 运行中的任务节点增加脉冲动画，失败节点高亮红色边框。
- **执行状态实时推送**：
  - 通过 SSE `/api/execution/{job_id}/status` 实时更新节点状态与日志。
  - 新增底部“运行控制台”抽屉，滚动显示 stdout/stderr。

### 4.5 Skills & Domains 功能补齐

- **Skill 管理**：
  - 列表展示：名称、版本、信任等级、状态（启用/禁用）、最近测试时间。
  - 操作：启用/禁用、锁定/解锁、运行测试、查看代码、升级 CodeAct 运行记录为 Skill。
  - 搜索与筛选：按领域标签、信任等级、状态筛选。
  - 导入：支持拖拽 `skill.zip` 或 YAML 文件。
- **Domain 市场**：
  - 卡片式展示可用 Domain 模板（单细胞、空间转录组、微生物组等）。
  - 每个模板显示：描述、包含 Skills、示例数据集、适用场景。
  - 操作：预览、导入、导出为 zip、一键创建项目。

### 4.6 报告与结果展示

- **报告面板升级**：
  - 支持 Markdown/HTML/Notebook 渲染。
  - 图表使用 Plotly/ECharts 渲染，支持缩放、导出 PNG/SVG。
  - 关联原始任务与数据文件，支持“查看来源”、“复现此结果”。
- **可复现包入口**：
  - 每个完成的任务提供“下载 ReproducibilityBundle”按钮。
  - Bundle 内含：执行代码、依赖环境、输入数据引用、运行参数、结果文件。

### 4.7 空状态与异常处理

- 为每个功能模块设计空状态插画/图标与引导文案：
  - 无会话：引导新建会话。
  - 无任务：提示输入分析需求。
  - 无 Skill：引导导入或生成。
  - 无 Domain：引导浏览市场。
- 统一错误提示：
  - 网络错误：重试按钮。
  - API 错误：展示后端返回的错误码与详情。
  - 执行错误：展示错误日志片段与“复制日志”按钮。

### 4.8 响应式与可访问性

- 支持平板/小屏自适应：左侧导航可折叠，聊天与工作区可切换全屏。
- 增加键盘快捷键提示（`?` 唤起）。
- 表单元素增加 `label`、`aria-*` 属性，支持屏幕阅读器。

## 5. 推荐技术栈补充

| 用途 | 推荐库 | 说明 |
|------|--------|------|
| UI 组件库 | `shadcn/ui` 或 `Radix UI` | 可定制、无样式污染、与 Tailwind 结合好 |
| 图标 | `lucide-react` | 统一、清晰、科研工具风格 |
| Markdown 渲染 | `react-markdown` + `remark-gfm` + `rehype-highlight` | 支持表格、代码高亮 |
| 公式 | `react-katex` | 支持科研 LaTeX 公式 |
| 图表 | `recharts` / `plotly.js` | 根据后端返回选择 |
| 代码编辑器 | `@monaco-editor/react` | 展示/编辑生成的代码 |
| 表格 | `@tanstack/react-table` | Skill/任务/Domain 列表 |
| 拖拽上传 | `react-dropzone` | 文件与 Skill/Domain 导入 |
| 虚拟滚动 | `react-window` | 长消息/日志列表 |
| 主题 | `next-themes` 或自实现 | 深色模式 |

## 6. 分阶段实施建议

### 第一阶段：基础体验（1–2 周）

- 搭建 `components/ui` 通用组件库（Button、Input、Select、Modal、Badge、Skeleton、EmptyState）。
- 全局布局重构：顶部栏 + 左侧导航 + 主题切换。
- 聊天消息气泡支持 Markdown、代码高亮、复制、重新生成。
- 空状态与错误提示统一处理。

### 第二阶段：配置与对话闭环（2 周）

- 新增 Settings 配置中心（模型、执行环境、预算）。
- 计划审批 UI（批准/修改/拒绝）。
- HITL 交互面板。
- 多会话管理。

### 第三阶段：工作流与执行监控（2 周）

- SSE 实时任务状态推送。
- 节点详情抽屉、日志控制台。
- 任务重试/跳过/中止操作。
- DAG 自动布局。

### 第四阶段：Skill & Domain 生态（2 周）

- Skill 列表/搜索/筛选/测试/锁定。
- Domain 市场卡片化、导入/导出/预览。
- 可复现包下载入口。

### 第五阶段：报告与 polish（1–2 周）

- 报告渲染升级（图表、Notebook）。
- 快捷键、响应式、可访问性。
- 性能优化（虚拟滚动、懒加载、组件拆分）。

---

## 7. 预期效果

- 前端从“功能可用”升级为“专业可用”，显著提升用户信任感。
- 配置中心降低使用门槛，让非技术用户也能快速上手。
- 计划审批、HITL、实时监控等功能补齐后，后端核心能力得到完整呈现。
- Skill 与 Domain 生态在前端有清晰入口，促进社区扩展与复用。
- 为后续商业化与个人用户付费版本打下体验基础。

## 8. 需要后端配合的接口（建议）

| 接口 | 用途 |
|------|------|
| `GET/PUT /api/settings` | 全局配置读写 |
| `GET /api/models` | 列出可用模型/Provider |
| `GET /api/sessions` | 会话列表 |
| `POST /api/plan/{id}/approve` | 批准计划 |
| `POST /api/plan/{id}/modify` | 修改计划 |
| `POST /api/plan/{id}/reject` | 拒绝计划 |
| `POST /api/chat/hitl/respond` | HITL 回复 |
| `GET /api/execution/{job_id}/status` | SSE 执行状态 |
| `POST /api/skills/{id}/test` | Skill 测试 |
| `POST /api/skills/{id}/lock` / `unlock` | Skill 锁定/解锁 |
| `GET /api/reproducibility/{task_id}/bundle` | 下载可复现包 |

---

## 9. 总结

当前 HomomicsLab 前端已经完成了核心功能的“骨架”，但要达到生产级、面向个人用户的生物信息 Agent 工具标准，必须在**视觉设计、配置能力、交互细节、功能完整度**四个维度进行系统升级。建议按阶段推进，优先补齐配置中心、聊天闭环和任务监控，再逐步完善 Skill/Domain 生态与报告体验。
